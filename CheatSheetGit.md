El commit se usa para guardar los cambios realizados en el proyecto. Primero se agregan los archivos con git add . y luego se crea el commit con git commit -m "mensaje del cambio".

El push sirve para enviar esos cambios al repositorio remoto usando git push origin main.

El pull se utiliza para descargar y actualizar el proyecto con los cambios del repositorio usando git pull origin main.

El flujo básico es: primero pull, luego hacer cambios, después commit y finalmente push.
